# 架空企業コーポレートサイト（表） & 管理画面（裏 / WP風）サイトマップ・構成

## 0. 前提

- 実装は任意（WordPressで構築する前提ではない）
- 管理画面は「WPっぽい情報設計・UI語彙・画面構造」を採用する
- ホラー体験ARGとして「表（公開）⇄ 裏（管理）」が1対1で対応し、同じ情報が別の意味に見える設計にする

---

## 1. 公開サイト（表）サイトマップ

### 1-1. 必須ページ（初期リリース最小）

- `/` トップ
- `/company` 会社概要（理念 / 沿革 / 会社情報）
- `/services` サービス（提供領域 / 依頼フロー）
- `/works` 実績（案件一覧 / 実績詳細）
- `/news` ニュース・記事（一覧 / 記事詳細）
- `/staff` 社員紹介（一覧 / 詳細）
- `/contact` お問い合わせ（フォーム）

---

## 2. 管理画面（裏 / WP風）サイトマップ

### 2-1. 左メニュー構造（WP風）

- ダッシュボード
- 投稿（Posts）
- 固定ページ（Pages）
- メディア（Media）
- コメント（Comments）
- フォーム受信箱（Inbox）※プラグインっぽい
- 実績（Works）※カスタム投稿タイプ
- スタッフ（Staff）※カスタム投稿タイプ（ユーザーとは別）
- 外観（Appearance）
- プラグイン（Plugins）
- ツール（Tools）
- 設定（Settings）

---

## 3. 表（公開）⇄ 裏（管理）の対応表

| 公開（表）     | URL例       | 管理（裏 / WP風）       | 画面の見え方（要点）                               |
| -------------- | ----------- | ----------------------- | -------------------------------------------------- |
| TOP            | `/`         | 固定ページ > ホーム     | ブロック編集＋右サイドバーに指標メタ（薄い異物）   |
| 会社概要       | `/company`  | 固定ページ > 会社概要   | 沿革ブロックに内部タグ（Sealed/Rewritten等）       |
| サービス       | `/services` | 固定ページ > サービス   | テンプレ/ブロック名にだけ異物（protocol等）        |
| 実績           | `/works`    | 実績（Works）           | 一覧テーブルに1列だけ異物（例：発現）              |
| ニュース・記事 | `/news`     | 投稿（Posts）           | 下書き・承認フロー・内部レビュー（叱責）           |
| 社員紹介       | `/staff`    | スタッフ（Staff）       | 削除済み/ Missing が見える。削除理由が内部のみ表示 |
| お問い合わせ   | `/contact`  | フォーム受信箱（Inbox） | 区分に「兆候」などが紛れる                         |

## 4. 画面設計ルール（WP風UIの型）

### 4-1. 一覧画面（List）

- 上部：検索 / 絞り込み（カテゴリ・タグ・日付）/ 一括操作
- タブ：`すべて / 公開 / 下書き / 非公開 / ゴミ箱`（必要に応じて追加）
- 中央：テーブル（列＋行）
- 行アクション：`編集 / クイック編集 / ゴミ箱へ / 表示`

### 4-2. 編集画面（Edit）

- メイン：ブロック編集（本文）
- 右：公開パネル（公開状態・可視性・公開日時・スラッグ）
- 下：メタボックス（レビュー/内部メモ/カスタムフィールド）

### 4-3. 異物（ホラー）差し込み規則

- 1画面に1〜2箇所まで
- 深部（削除済み・権限限定・詳細）ほど濃く
- ラベル置換は段階的に（例：公開→放流、承認→封印 は後半で出す）

---

## 5. 主要コンテンツタイプ（データの単位）

### 5-1. 投稿（Posts）＝ニュース・記事

- `status`: draft / pending / publish / rejected / (拡張) leak
- `review_comments`: 内部レビューコメント（叱責が出る場所）
- `visibility`: public / private
- `meta`: 露出度・定着（β）等（後から足せる）

### 5-2. 固定ページ（Pages）＝TOP/会社概要/サービス/社員紹介TOP/問い合わせ

- ブロック構成中心
- `template`: ページテンプレ（ここに異物名を混ぜられる）

### 5-3. 実績（Works）＝案件

- `client`, `venue`, `date`, `status`
- `status`例：planned / live / closed / (拡張) sealed / rewritten
- 1列だけ異物（例：発現 OFF/LOW/MID/HIGH）

### 5-4. スタッフ（Staff）＝社員紹介

- `name`, `role`, `team`, `bio`, `photo`, `visibility`
- `state`例：active / suspended / (拡張) missing
- `removed_reason`（削除済みでのみ表示）
  - 例：`〇〇の顕現実験により死亡（MNF-02）`

### 5-5. フォーム受信箱（Inbox）＝お問い合わせ

- `category`：一般 / 取材 / 見積 / クレーム / (拡張) 兆候
- `severity`：low / medium / high
- `status`：open / pending / resolved / (拡張) rewritten
